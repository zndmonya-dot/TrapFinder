# プロプランの採算性分析

## OpenAI APIコスト（2024年12月時点）

### GPT-4o
- **Input**: $2.50 / 1M tokens
- **Output**: $10.00 / 1M tokens

### GPT-4o-mini
- **Input**: $0.15 / 1M tokens
- **Output**: $0.60 / 1M tokens

**コスト差**: GPT-4oはGPT-4o-miniの約**16.7倍**

## 現在のプロプラン設定

- **価格**: ¥980/月
- **GPT-4o使用回数**: 1日20回まで
- **文字数制限**: 50,000文字/回
- **制限到達後**: 自動でGPT-4o-miniに切り替え

## コスト計算

### 前提条件
- 1文字 ≈ 0.25トークン（日本語の場合）
- 50,000文字 ≈ 12,500トークン（入力）
- 平均的な出力: 5,000トークン（推定）

### GPT-4oの1回あたりコスト
- **Input**: 25,000 / 1,000,000 × $2.50 = **$0.0625**
- **Output**: 5,000 / 1,000,000 × $10.00 = **$0.05**
- **合計**: **$0.1125** ≈ **¥17**（$1=¥150として）

### 月間最大コスト
- 1日20回 × 30日 = **600回/月**（最大）
- 600回 × ¥17 = **¥10,200/月**（最大コスト）

### 採算性の問題点
- **現在の価格**: ¥980/月
- **最大コスト**: ¥10,200/月
- **問題**: ユーザーが上限まで使用すると、**約10倍のコスト**が発生

## 改善案

### 案1: 使用回数を減らす（推奨）
- **1日10回まで**に変更
- 月間最大コスト: ¥5,100/月
- まだ採算が厳しいが、改善される

### 案2: 価格を上げる
- **¥2,980/月**に変更
- 月間最大コスト: ¥10,200/月
- マージン: 約70%（Apple手数料30%考慮）

### 案3: 文字数制限を下げる
- **50,000文字/回**に変更
- 1回あたりコスト: 約¥8.5
- 月間最大コスト: ¥5,100/月（1日20回の場合）

### 案4: 組み合わせ（推奨）
- **1日10回まで** + **50,000文字/回**
- 月間最大コスト: 約¥2,550/月
- 価格: **¥1,980/月**（適正価格）

### 案5: トークン制限を追加
- **1日あたり500,000トークンまで**（入力+出力）
- これにより、長文解析のコストを制限

## 実際の使用パターンを考慮

### 想定される使用パターン
1. **軽度ユーザー**: 1日1-3回 → コスト: ¥51-153/月
2. **中程度ユーザー**: 1日5-10回 → コスト: ¥255-510/月
3. **重度ユーザー**: 1日15-20回 → コスト: ¥765-1,020/月

### 平均的な使用率
- ユーザーの80%は1日10回未満と仮定
- 平均使用回数: 1日5回程度
- 平均コスト: 約¥255/月

## 結論

**現在の設定（¥980/月、1日20回）は採算が合いません。**

### 推奨される変更
1. **価格を¥1,980-2,980/月に引き上げ**
2. **使用回数を1日10-15回に減らす**
3. **または、文字数制限を50,000文字/回に下げる**

最もバランスが良いのは**オプションB（¥2,980/月、1日15回）**です。
