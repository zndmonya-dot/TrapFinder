# スタンダードプランの採算性分析

## 現在のスタンダードプラン設定

- **価格**: ¥480/月
- **AIモデル**: GPT-4o-mini
- **スキャン回数**: **無制限**
- **文字数制限**: 50,000文字/回

## OpenAI APIコスト（GPT-4o-mini）

- **Input**: $0.15 / 1M tokens
- **Output**: $0.60 / 1M tokens

## コスト計算

### 前提条件
- 1文字 ≈ 0.25トークン（日本語の場合）
- 50,000文字 ≈ 12,500トークン（入力）
- 平均的な出力: 5,000トークン（推定）

### GPT-4o-miniの1回あたりコスト
- **Input**: 25,000 / 1,000,000 × $0.15 = **$0.00375**
- **Output**: 5,000 / 1,000,000 × $0.60 = **$0.003**
- **合計**: **$0.00675** ≈ **¥1**（$1=¥150として）

### 月間コストのシミュレーション

#### ケース1: 軽度ユーザー（1日3回）
- 1日3回 × 30日 = 90回/月
- 90回 × ¥1 = **¥90/月**
- **マージン**: ¥480 - ¥90 = **¥390/月**（約81%）

#### ケース2: 中程度ユーザー（1日10回）
- 1日10回 × 30日 = 300回/月
- 300回 × ¥1 = **¥300/月**
- **マージン**: ¥480 - ¥300 = **¥180/月**（約38%）

#### ケース3: 重度ユーザー（1日30回）
- 1日30回 × 30日 = 900回/月
- 900回 × ¥1 = **¥900/月**
- **マージン**: ¥480 - ¥900 = **-¥420/月**（**赤字**）

#### ケース4: 極端なケース（1日100回）
- 1日100回 × 30日 = 3,000回/月
- 3,000回 × ¥1 = **¥3,000/月**
- **マージン**: ¥480 - ¥3,000 = **-¥2,520/月**（**大幅な赤字**）

## 問題点

### 1. 無制限のリスク
- **無制限**のため、悪用や過度な使用でコストが無限に増加する可能性
- 1日100回使用されると、月間で¥3,000のコストが発生
- 価格¥480に対して、**約6倍のコスト**が発生

### 2. Apple手数料30%を考慮
- 実質受取額: ¥480 × 0.7 = **¥336/月**
- ケース3（1日30回）: ¥336 - ¥900 = **-¥564/月**（**赤字**）

## 改善案

### 案1: 使用回数制限を追加（推奨）
- **1日50回まで**に制限
- 月間最大コスト: ¥1,500/月
- 価格: ¥480/月
- **問題**: まだ採算が厳しい（1日50回使用で赤字）

### 案2: 価格を上げる
- **¥980/月**に変更
- 1日50回使用でも: ¥980 × 0.7 - ¥1,500 = **-¥814/月**（まだ赤字）
- 1日30回使用で: ¥980 × 0.7 - ¥900 = **-¥214/月**（まだ赤字）

### 案3: 使用回数制限 + 価格調整（推奨）
- **価格**: ¥680/月
- **使用回数**: 1日30回まで
- 月間最大コスト: ¥900/月
- 実質受取額: ¥680 × 0.7 = ¥476/月
- **問題**: まだ採算が厳しい

### 案4: 文字数制限を下げる
- **50,000文字/回**に変更
- 1回あたりコスト: 約¥0.5
- 1日50回使用: ¥750/月
- 価格: ¥480/月
- **問題**: まだ採算が厳しい

### 案5: 組み合わせ（最も推奨）
- **価格**: ¥680/月
- **使用回数**: 1日30回まで
- **文字数制限**: 50,000文字/回
- 1回あたりコスト: 約¥0.5
- 月間最大コスト: ¥450/月（1日30回 × ¥0.5 × 30日）
- 実質受取額: ¥680 × 0.7 = **¥476/月**
- **マージン**: ¥476 - ¥450 = **¥26/月**（約5%）

## 推奨設定

### オプションA: コスト重視（推奨）
- **価格**: ¥680/月
- **使用回数**: 1日30回まで
- **文字数制限**: 50,000文字/回
- **月間最大コスト**: ¥450/月
- **マージン**: 約5%（Apple手数料30%考慮）

### オプションB: バランス型
- **価格**: ¥980/月
- **使用回数**: 1日50回まで
- **文字数制限**: 50,000文字/回（現状維持）
- **月間最大コスト**: ¥1,500/月
- **マージン**: 約-16%（Apple手数料30%考慮）
- **問題**: まだ採算が厳しい

### オプションC: 現状維持 + 使用回数制限
- **価格**: ¥480/月（現状維持）
- **使用回数**: 1日20回まで
- **文字数制限**: 50,000文字/回（現状維持）
- **月間最大コスト**: ¥600/月
- **実質受取額**: ¥480 × 0.7 = ¥336/月
- **マージン**: ¥336 - ¥600 = **-¥264/月**（**赤字**）

## 実際の使用パターンを考慮

### 想定される使用パターン
1. **軽度ユーザー**: 1日1-5回 → コスト: ¥30-150/月 → **採算OK**
2. **中程度ユーザー**: 1日10-20回 → コスト: ¥300-600/月 → **採算が厳しい**
3. **重度ユーザー**: 1日30回以上 → コスト: ¥900+/月 → **赤字**

### 平均的な使用率
- ユーザーの70%は1日10回未満と仮定
- 平均使用回数: 1日8回程度
- 平均コスト: 約¥240/月
- **採算**: ¥480 × 0.7 - ¥240 = ¥96/月（約20%マージン）

## 結論

**現在の設定（¥480/月、無制限）はリスクが高いです。**

### 推奨される変更
1. **使用回数を1日30回までに制限**
2. **価格を¥680/月に引き上げ**（または¥480/月のままでも可）
3. **文字数制限を50,000文字/回に下げる**（オプション）

最もバランスが良いのは**オプションA（¥680/月、1日30回、50,000文字/回）**です。

ただし、**ユーザー体験を重視する場合は、¥480/月のまま1日30回まで**でも、平均的な使用パターンでは採算が取れます。
